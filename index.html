Housify

Rental Web App

Jan 2021 – Dec 2021

Description

Creating my custom rental web application.

Many people have trouble finding a place to live because there is no unified system for housing in Latvia. Instead there are more than 10 different websites competing for the main position in housing industry. Each of those websites have their own housing posters, their own partners and unique web-page layout. My application collects all posters from these websites and displays unified rent / sale housing posters in simple layout.

System uses microservice architecture. Several docker pods run independant services and communicate with others. Architecture can be seen in the added graph containing application structure.

Microservice side uses Java Spring code running on Tomcat servers. WebSockets for instant client-server communication / updates. Posters are loaded from the PostgreSQL database and cached on server-side to improve performance.

Custom crawler process runs on separate Java application, periodically crawls all websites, finds new housing posters and stores them into the database. Crawler uses JSoup to read / parse the HTML contents of each website.

Advanced features include viewing statistics for specific address, viewing historic posters in the same address and viewing global statistics for specific time period.

Technologies
  • Back-End - Linux Tomcat WebServer & Java code
  • Front-End - Responsive Front-End using HTML, CSS, JS, JQuery
  • WebSockets - WebSockets for fast client / server communication
  • WebCrawler - Automatic web crawler / parser for multiple websites
  • PostgreSQL - Large scale data storage with PostgreSQL + Hibernate

DigiBots

Multiplayer Game

May 2020 – Dec 2020

Description

Game about building your bots and competing against other players in epic bot-battles.

Each player can have up to 4 bots, Each bot consists of multiple parts. Each part is different and grants different stats & bonuses. This allows for great customization and variety of bots.

Bot battles happen as simulations inside an arena. Players place the bots one-by-one and choose bot path. Bots will continue to move along the chosen path. Bots will automatically aim and fire all equipped weapon parts. Bots use fully functional 3D physics for extremely realistic battles.

The game has partial multiplayer support - All player & bot data is stored and synchronized on Java Tomcat server. Battles load initial bot data from server and continue to run as simulations.

Game also has item packs - containers that can be purchased as micro-transactions and contain several parts.

Game world has several iterations - one prototype is a 3D cyberspace world with ability to enter various different cyber spaces. Another prototype is a lobby-style world where players have access to all actions.

Game was originally designed for mobile phones, but development was paused due to performance issues on mobile devices.

Technologies

  • Multiplayer - Custom TCP multiplayer Java server

  • Visuals - Lighting, Textures, Animations, Particles
  • Physics - Fully functional 3D physics implementation
  • Scripting - Game Mechanics, Movement, Transitions
